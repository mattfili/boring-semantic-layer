## Data Storage

**Databases (via ibis):**
- **DuckDB** - Primary in-memory and file-based backend
  - Connection: Profile-based config with `type: duckdb`
  - Client: ibis via xorq vendored version
  - Used for: Examples, tests, development
  - Configuration: `examples/profiles.yml` defines `example_db` profile

- **Other ibis Backends** - Pluggable via profile system
  - PostgreSQL, BigQuery, Snowflake, Redshift, etc.
  - Connection: Profile-based via `get_connection()` at `src/boring_semantic_layer/profile.py`
  - Client: ibis abstracts all database connections

**File Storage:**
- **Parquet Files** - Data source format
  - Source: Remote URLs (S3/R2 compatible) or local files
  - Example data location: `https://pub-a45a6a332b4646f2a6f44775695c64df.r2.dev/`
  - Loaded via: `connection.read_parquet()` with table registration
  - Examples in: `examples/profiles.yml` (6 parquet files for flights, carriers, airports)

**Caching:**
- **Prompt Caching** - Anthropic-specific optimization
  - Implementation: `AnthropicPromptCachingMiddleware` at `src/boring_semantic_layer/agents/backends/langgraph.py` line 85
  - Reduces token usage for repeated prompts

## MCP Server Implementation

**FastMCP 3.0 Compatibility (Current: 2.12.4+):**

**Current Implementation:**
- Location: `src/boring_semantic_layer/agents/backends/mcp.py`
- Class: `MCPSemanticModel(FastMCP)` - Extends FastMCP base class
- Tool Registration: Decorator-based via `@self.tool()` pattern
- Prompts: Loaded from `docs/md/prompts/query/mcp/` with dynamic path resolution

**Tool Definitions (5 tools):**
1. **list_models()** - Returns available semantic models
   - Description: Loaded from `tool-list-models-desc.md`
   - No parameters
   - Returns: `{model_name: description}`

2. **get_model(model_name)** - Detailed model schema
   - Description: Loaded from `tool-get-model-desc.md`
   - Returns: Dimensions, measures, calculated measures with metadata
   - Implementation: Lines 67-101

3. **query_model(model_name, dimensions, measures, filters, order_by, limit, time_grain, time_range, get_records, records_limit, get_chart, chart_backend, chart_format, chart_spec)**
   - Description: Execute semantic queries with optional chart generation
   - Parameters use Pydantic validators (`BeforeValidator(_parse_json_string)`)
   - Handles JSON string parsing for complex types
   - Generates charts via `generate_chart_with_data()`
   - Implementation: Lines 133-261

4. **get_time_range(model_name)** - Time dimension bounds
   - Description: Loaded from `tool-get-time-range-desc.md`
   - Returns: `{"start": ISO date, "end": ISO date}`
   - Handles joined models with column prefix parsing
   - Implementation: Lines 103-131

5. **search_dimension_values(model_name, dimension_name, search_term, limit)**
   - Description: Autocomplete for dimension values
   - Case-insensitive regex search with frequency counts
   - Fallback to top values if no matches
   - Implementation: Lines 263-352

**System Instructions:**
- Loaded from `docs/md/prompts/query/mcp/system.md`
- Customizable via constructor parameter
- Fallback to default: "MCP server for semantic models"

**MCP Connection:**
- Uses `server.run()` to start MCP protocol listener
- No explicit socket/host configuration required (FastMCP handles)
- Configuration via `~/.config/Claude/claude_desktop_config.json` for Claude Desktop
- Example config shows `uv run example_mcp.py` command

**Data Format:**
- Tool parameters: Pydantic-based with JSON schema generation
- Tool outputs: JSON strings (query results, model schemas)
- Chart data: Embedded in query results as JSON with data + spec
- Error handling: Raises ValueError which becomes MCP error responses

**Examples:**
- `examples/example_mcp.py` - Flight data semantic layer server
- `examples/example_mcp_cohort.py` - Cohort analysis with joined models

**Testing:**
- Unit tests: `src/boring_semantic_layer/agents/tests/test_semantic_mcp.py`
- JSON parsing: `src/boring_semantic_layer/tests/test_mcp_json_parsing.py`
- Integration: Tests validate tool schema and execution

## Lazy Loading & Optional Dependencies

**Import Strategy:**
- Core: `ibis`, `pydantic`, `xorq`, `returns`, `toolz`, `attrs`, `pyyaml`
- Optional `[mcp]`: `fastmcp`
- Optional `[agent]`: `langchain`, `langchain-anthropic`, `langchain-openai`, `python-dotenv`
- Optional `[examples]`: `xorq[duckdb]`, `duckdb`
- Optional `[viz-*]`: `altair`, `plotly`, `plotext`

**Lazy Import Pattern:**
- `__getattr__()` in `__init__.py` raises ImportError with helpful message if extra not installed
- Example: `MCPSemanticModel` requires `[mcp]` extra
- Example: `LangGraphBackend` requires `[agent]` extra
- Prevents dependency bloat for base installation
